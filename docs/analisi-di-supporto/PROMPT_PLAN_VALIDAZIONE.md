# Prompt per Modalità Plan - Validazione Analisi Retrospettiva

## Prompt da usare in Cursor Plan Mode

```
Analizza e valida l'analisi retrospettiva dei progetti completati (mscarichi, cactusdashboard, klabhouse, msmeatdashboard) per calibrare le stime del preventivo L'Altramusica.

CONTESTO:
- Ho già completato un'analisi retrospettiva che confronta 4 progetti Laravel completati
- L'analisi ha calcolato ore di lavoro, linee di codice (LOC), e ratio LOC/ora
- Il risultato suggerisce di ridurre le ore del preventivo L'Altramusica da 980 a 725 ore (-26%)

OBIETTIVO:
Creare un piano di VALIDAZIONE (non sviluppo) per confermare che l'analisi retrospettiva sia:
1. Metodologicamente corretta
2. Statisticamente significativa
3. Difendibile con il cliente
4. Completa e non tralascia fattori importanti

REQUISITI DEL PIANO:
- Identificare punti deboli/assunzioni dell'analisi attuale
- Proporre verifiche aggiuntive da fare
- Suggerire metriche di validazione incrociata
- Definire criteri di accettazione per confermare i risultati
- Identificare rischi e limiti dell'analisi
- Proporre miglioramenti metodologici

NON includere:
- Piano di sviluppo software
- Implementazione di funzionalità
- Codice o modifiche al sistema

INCLUDE:
- Checklist di validazione
- Verifiche da fare sui dati
- Confronti aggiuntivi possibili
- Analisi di sensibilità
- Documentazione per difendere l'analisi

Il piano deve essere eseguibile e permettere di confermare o correggere le stime finali.
```

---

## Note per l'uso

1. Copia il prompt sopra
2. Attiva la modalità Plan in Cursor
3. Incolla il prompt
4. Cursor genererà un piano strutturato di validazione

Il piano risultante ti aiuterà a:
- Verificare che l'analisi sia corretta
- Identificare eventuali correzioni necessarie
- Rendere l'analisi più robusta e difendibile
- Avere una checklist per validare i risultati
